一、项目介绍 📖

这是一个前后端分离的后端服务项目，基于 Spring Boot 框架开发，核心围绕三大生活化场景搭建：

- 家具商城 🛋️：实现基础商品展示、购物车管理、简单下单等电商核心逻辑（目前为初级实现版本）；

- 智能家居控制 🏡：聚焦模拟设备连接、基础指令下发（比如灯光开关、空调温度调节），暂未对接真实硬件；

- 社交互动 🗣️：支持用户关注、简单动态发布功能，主打家具使用体验分享与智能生活经验交流。

当前项目处于初期开发阶段，代码存在不少可优化空间，后续会持续迭代完善功能~

三、技术栈选型 🛠️

（初学者友好型技术组合，以实用易上手为核心原则）

- 核心框架：Spring Boot 2.x

- 数据库：MySQL 8.0

- ORM 框架：MyBatis-Plus（简化CRUD操作，降低入门难度）

- 认证授权：JWT（JSON Web Token）

- API 文档：Swagger/knife4j（接口可视化，方便调试）

- 开发工具：IntelliJ IDEA + Git

四、本地运行步骤 🚀

1. 克隆仓库到本地：git clone [仓库地址]（将 [仓库地址] 替换为实际仓库链接）

2. 打开 IntelliJ IDEA，导入项目，等待 Maven 自动下载完成依赖 📥

3. 新建 MySQL 数据库，执行项目 sql/ 目录下的数据库初始化脚本

4. 找到 src/main/resources/application.yml 文件，修改数据库连接配置（替换为自己的MySQL用户名、密码、数据库名）

5. 定位到项目启动类（xxxApplication.java），右键运行启动 Spring Boot 项目

6. 项目启动成功后，访问 API 文档：http://localhost:8080/doc.html（默认端口为8080，若修改过端口需对应调整）

五、项目结构 📂

src/main/java/[包名]/
├── config/       // 配置类（Swagger、JWT、MyBatis等相关配置）
├── controller/   // 接口控制器（用户、商品、设备、动态等模块接口）
├── service/      // 业务逻辑层（核心业务处理）
├── mapper/       // Mapper接口（数据库操作相关）
├── model/        // 实体类、DTO、VO封装
│   ├── entity/   // 数据库表对应实体
│   ├── dto/      // 入参数据传输对象
│   └── vo/       // 出参数据封装对象
├── exception/    // 全局异常处理相关
└── [项目名]Application.java  // 项目启动类
src/main/resources/
├── application.yml  // 全局配置文件（数据库、端口等）
└── sql/             // 数据库初始化脚本目录
